# Метрики качества персонализированного поиска

Данный документ обосновывает выбор метрик для оценки качества системы умного поиска СТЕ в соответствии с критериями хакатона (10 баллов).

---

## 1. Метрики качества ранжирования

### 1.1 NDCG@K (Normalized Discounted Cumulative Gain)

**Формула:** NDCG@K = DCG@K / IDCG@K, где DCG@K = Σ (rel_i / log2(i+1))

**Почему выбрана:**
- Главная метрика для задач ранжирования с учетом позиции в выдаче.
- Учитывает, что результат на позиции 1 ценнее, чем на позиции 10.
- Нормализована в диапазон [0, 1], удобна для сравнения.
- Стандарт de facto для Learning-to-Rank задач (именно наш случай с CatBoost Ranker).

**Наши K:** NDCG@5 (топ-5 для быстрых закупщиков) и NDCG@20 (полная страница).

**Релевантность (rel_i) определяется:**
- 3 — пользователь заключил контракт на этот СТЕ
- 2 — СТЕ из категории, по которой у пользователя есть контракты
- 1 — семантически похожий СТЕ
- 0 — нерелевантный результат

---

### 1.2 MRR (Mean Reciprocal Rank)

**Формула:** MRR = (1/N) Σ (1 / rank_i)

**Почему выбрана:**
- Измеряет, как быстро пользователь находит **первый** релевантный результат.
- Критична для сценария "введи запрос и немедленно найди нужный СТЕ" — именно то, что нужно закупщику.
- Особенно важна при работе с омографами (например, "ручка" → нужен правильный тип сразу).

---

### 1.3 MAP@K (Mean Average Precision)

**Формула:** MAP@K = (1/N) Σ AP@K_i, где AP@K = (1/rel_total) Σ P(k) * rel(k)

**Почему выбрана:**
- Оценивает точность по всей выдаче, а не только на топовых позициях.
- Хорошо работает при множественно-релевантных результатах (у СТЕ несколько возможных аналогов).

---

## 2. Метрики персонализации

### 2.1 Персонализированный NDCG (pNDCG)

NDCG считается отдельно для каждого пользователя (по его профилю), затем усредняется.

**Почему:** Стандартный NDCG предполагает глобальную релевантность. Наша система персонализированная — для закупщика из Образования релевантность другая, чем для Здравоохранения. pNDCG это учитывает.

---

### 2.2 History Hit Rate (HHR@K)

**Формула:** HHR@K = |{STE в топ-K} ∩ {STE из истории контрактов пользователя}| / K

**Почему:** Прямо измеряет, насколько хорошо система продвигает товары, которые пользователь уже покупал. Бизнес-метрика: повторные закупки — самый частый сценарий на Портале поставщиков.

---

### 2.3 Negative Signal Precision (NSP)

**Формула:** доля товаров, помеченных как "bounce/hide", которые система впоследствии понижает в выдаче.

**Почему:** Прямо измеряет качество учёта негативных сигналов — отдельный критерий оценки хакатона (25 баллов за персонализацию включают негативные сигналы).

---

### 2.4 Session Re-ranking Delta (SRD)

**Формула:** NDCG после N событий в сессии − NDCG до первого события.

**Почему:** Измеряет эффект динамической индексации — насколько выдача улучшается по мере накопления сигналов в рамках одной сессии. Прямо соответствует требованию ТЗ: "система должна перестраивать выдачу по мере накопления данных о поведении конкретного заказчика".

---

## 3. Метрики качества NLP

### 3.1 Typo Correction Rate (TCR)

Доля запросов с опечатками, для которых система предложила верное исправление.

### 3.2 Synonym Coverage Rate (SCR)

Доля синонимичных запросов (например, "ноутбук" / "лэптоп"), которые возвращают пересекающийся топ-10 результатов.

| SRD | Динамика | ВЫСОКИЙ | Эффект динамической индексации |
