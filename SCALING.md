# План масштабирования системы

## Текущий уровень (Хакатон / MVP)

```
1 сервер Ubuntu (France)
├── PostgreSQL 16 + pgvector + pg_trgm  (1 инстанс)
├── Redis 7                              (1 инстанс)
├── FastAPI + Uvicorn                    (1 процесс, 4 воркера)
└── Nginx → React SPA                   (1 инстанс)
```

Ожидаемая нагрузка: до 50 RPS, < 200 мс P95 latency.

---

## Уровень 2: Рост портала (до 1000 RPS)

**Что меняем:**
- FastAPI: увеличить число воркеров, добавить Gunicorn в front of Uvicorn.
- PostgreSQL: добавить **read replica** — все SELECT поиска идут в реплику, запись событий в master.
- Redis: перейти на **Redis Sentinel** (HA, failover).
- ML-модель: кэшировать предрасчитанные эмбеддинги СТЕ в pgvector при изменении каталога, не считать их при каждом запросе.
- Индексы: партиционировать таблицу `events` по `created_at` (RANGE), старые партиции архивировать.

```
Load Balancer (Nginx)
    ├── FastAPI × N воркеров
    ├── PostgreSQL master  ←→  read replica
    └── Redis Sentinel
```

## Уровень 3: Платформенный масштаб (портал поставщиков, > 10 000 RPS)

**Что меняем:**

| Компонент | Решение |
|---|---|
| Поиск | Выделенный **Elasticsearch/OpenSearch** кластер с кастомным плагином ранжирования |
| Векторный поиск | Переход с pgvector на **Milvus** или **Qdrant** (distributed vector DB) |
| ML-инференс | Выделенный **Triton Inference Server** для CatBoost + rubert-tiny2 (батчинг) |
| Персонализация | **Apache Kafka** для потока событий → фоновый пересчёт профилей |
| Кэш профилей | Redis Cluster (шардирование по user_inn) |
| Оркестрация | **Kubernetes** (HPA по CPU/RPS, rolling deploys) |

**Пересчёт профилей:**  
Сейчас: синхронный SQL при каждом поисковом запросе.  
На масштабе: Kafka consumer группа перестраивает профиль асинхронно каждые N минут → запись в Redis. Поиск читает готовый профиль из кэша, латентность запроса не растет.

## Уровень 4: Следующие доработки продукта

1. **A/B тестирование ранжирования** — разные версии CatBoost-модели для разных сегментов пользователей с онлайн-сравнением метрик (NDCG, CTR).
2. **Автоматическое переобучение** — CatBoost Ranker переобучается еженедельно на новых данных событий (click, bounce, contract) без ручного вмешательства.
3. **Расширение профиля** — добавить бюджет закупки, сезонность, географическую близость поставщика.
4. **Федеративное обучение** — без передачи сырых данных закупщиков в центр.
5. **Объяснимость 2.0** — SHAP-значения для каждой фичи CatBoost → детальный drill-down объяснений в UI.
